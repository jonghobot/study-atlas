DeepSeek V4를 이해하기 위한 관련 연구 리뷰

각 연구를 “V4가 왜 이런 구조를 택했는가?”라는 관점에서 요약합니다.

1. DeepSeek 직접 계보

2024
MoE

DeepSeekMoE: Towards Ultimate Expert Specialization in Mixture-of-Experts Language Models

요약
MoE에서 expert가 서로 다른 지식을 맡도록 fine-grained expert segmentation과 shared expert isolation을 제안합니다.
V4 연결
V4-Pro의 1.6T total / 49B active, V4-Flash의 284B / 13B 구조를 이해하는 핵심 배경입니다.
읽을 포인트
전체 용량을 키우면서 token당 계산량을 제한하는 전략이 어떻게 성능-비용 tradeoff를 바꾸는지.
한계/질문
MoE serving은 routing, load balancing, expert parallelism 때문에 배포 복잡도가 큽니다.
2025
DSA

DeepSeek-V3.2: Pushing the Frontier of Open Large Language Models

요약
DeepSeek Sparse Attention으로 long-context 상황의 attention 계산량을 줄이는 방향을 제시합니다.
V4 연결
V4의 CSA/HCA hybrid attention은 DSA 계열 문제의식, 즉 긴 context에서 dense attention이 너무 비싸다는 문제를 이어받습니다.
읽을 포인트
희소 attention이 품질 저하 없이 long-context 효율을 낼 수 있는 조건.
한계/질문
sparse selection이 중요한 먼 문맥을 놓칠 가능성을 어떻게 줄일 것인가.
2025
reasoning

DeepSeek-R1 계열 reasoning model

요약
reasoning 능력을 강화한 오픈 모델로, 비용 대비 성능과 공개 전략이 큰 주목을 받았습니다.
V4 연결
V4의 Thinking / Non-Thinking 모드를 이해하는 제품적·기술적 배경입니다.
읽을 포인트
추론 능력은 base architecture뿐 아니라 post-training, RL, distillation에 크게 좌우됩니다.
한계/질문
긴 추론 trace와 긴 external context가 결합될 때 hallucination/overthinking을 어떻게 제어할 것인가.

2. Long-context 확장 연구

2023
RoPE

YaRN: Efficient Context Window Extension of Large Language Models

요약
RoPE 기반 모델의 context window를 효율적으로 늘리는 방법을 제안합니다.
V4 연결
YaRN은 기존 모델의 context 길이 확장, V4는 초장문맥 비용 구조 자체의 개선이라는 차이가 있습니다.
핵심 질문
긴 위치 인코딩을 잘 외삽해도, attention/KV cache 비용은 여전히 남습니다.
활용
V4와 다른 long-context 모델을 비교할 때 “context extension”과 “efficient long-context inference”를 분리해 생각하게 해줍니다.
2023
fine-tuning

LongLoRA: Efficient Fine-tuning of Long-Context Large Language Models

요약
long-context LLM을 비교적 저렴하게 fine-tuning하기 위한 방법입니다.
V4 연결
V4를 특정 기업/도메인 장문맥 작업에 맞게 적응시키려면 이런 fine-tuning 비용 절감 연구가 중요합니다.
읽을 포인트
학습 시 sparse pattern과 추론 시 dense/global behavior의 간극.
한계/질문
1M context급으로 갈수록 fine-tuning 데이터와 평가셋 구성 자체가 병목입니다.
2023
systems

Ring Attention with Blockwise Transformers for Near-Infinite Context

요약
attention 계산을 blockwise로 나누고 여러 device에 ring 형태로 분산해 매우 긴 context를 처리합니다.
V4 연결
V4가 모델 구조를 통해 비용을 줄인다면, Ring Attention은 시스템 병렬화로 context 한계를 밀어붙입니다.
읽을 포인트
long-context는 모델 연구이면서 동시에 serving system 연구입니다.
한계/질문
분산 통신 비용과 실제 latency가 제품 경험에서 얼마나 감당되는가.
2024
streaming

StreamingLLM: Efficient Streaming Language Models with Attention Sinks

요약
attention sink 현상을 이용해 고정 window 모델도 긴 스트리밍 입력에서 안정적으로 동작하게 합니다.
V4 연결
에이전트가 오래 실행될 때 모든 과거를 다 보는 것보다 무엇을 유지할지 결정하는 문제가 중요합니다.
읽을 포인트
긴 context capacity와 지속적 streaming memory는 다른 문제입니다.
한계/질문
초기 token 보존만으로 복잡한 repo/document memory를 충분히 다룰 수 있는가.

3. Attention/KV 효율 연구

2022
kernel

FlashAttention: Fast and Memory-Efficient Exact Attention with IO-Awareness

요약
GPU memory hierarchy를 고려해 exact attention을 더 빠르고 메모리 효율적으로 계산합니다.
V4 연결
V4의 CSA/HCA가 algorithm/architecture 차원의 효율화라면, FlashAttention은 implementation/kernel 차원의 기반입니다.
읽을 포인트
LLM 효율은 수식 복잡도뿐 아니라 메모리 이동 비용에 의해 좌우됩니다.
한계/질문
exact attention 최적화만으로 1M context serving을 충분히 해결하기는 어렵습니다.
2024–2026
KV cache

KV Cache Compression 연구군

요약
long-context inference에서 KV cache가 메모리 병목이 되므로 token/head/layer 단위로 cache를 줄이는 연구가 활발합니다.
V4 연결
V4 발표에서 강조한 “KV cache 부담 감소”는 이 연구 흐름의 제품화된 핵심 주장입니다.
읽을 포인트
중요한 과거 정보를 버리지 않으면서 cache를 줄이는 기준이 무엇인가.
한계/질문
요약/압축이 reasoning fidelity와 source attribution을 얼마나 해치는가.

4. Agentic Coding 관련 연구/평가

Benchmark
agent

SWE-bench / repo-level coding benchmarks

요약
실제 repository issue를 고치고 test를 통과하는지를 평가하는 흐름입니다.
V4 연결
DeepSeek V4가 말하는 “agentic coding”은 단답 코딩이 아니라 repo 이해, test 로그 해석, 장기 tool-use loop와 관련됩니다.
읽을 포인트
긴 context가 실제 bug fixing 성공률을 높이는지, 아니면 retrieval/tool design이 더 중요한지.
한계/질문
benchmark 점수가 실제 개발자 workflow 품질과 얼마나 일치하는가.
