DeepSeek V4 관련 연구 지도

DeepSeek V4는 갑자기 나온 단일 모델이 아니라, MoE scaling, sparse/efficient attention, long-context extension, KV cache optimization, agentic coding 평가 흐름 위에 있습니다.

1. DeepSeek 계열 직접 선행 연구

연구관계DeepSeek V4와의 연결
DeepSeekMoE (2024)MoE architecturefine-grained expert segmentation과 shared expert isolation으로 expert specialization을 높이는 구조. V4-Pro/Flash의 “전체 파라미터는 크지만 active 파라미터는 작다”는 효율 전략의 기반.
DeepSeek-V3 / V3.1open LLM scaling대규모 MoE, FP8 학습/추론, 저비용 학습 전략 등 DeepSeek 계열의 성능-비용 균형을 축적한 전 단계.
DeepSeek-V3.2 / DSA (2025)sparse attentionDeepSeek Sparse Attention으로 long-context 계산량을 줄이는 방향. V4의 CSA/HCA hybrid attention으로 이어지는 직접적인 기술적 선행.
DeepSeek-R1 (2025)reasoning/post-trainingreasoning mode, RL 기반 추론 능력 강화, open reasoning model의 시장 충격. V4의 thinking mode 이해에 필요한 배경.

2. Long-context / Efficient Attention 연구

연구핵심V4와 비교 포인트
FlashAttention (2022)IO-aware exact attention으로 attention 계산의 메모리 병목을 줄임.V4가 attention 구조 자체를 압축/희소화한다면, FlashAttention은 kernel/IO 최적화 쪽의 기반 기술.
YaRN (2023)RoPE 기반 모델의 context window를 효율적으로 확장.“기존 모델을 더 긴 context로 늘리는 법”과 “처음부터 long-context 효율 구조를 설계하는 법”의 차이를 보여줌.
LongLoRA (2023)shifted sparse attention과 LoRA로 long-context fine-tuning 비용을 낮춤.학습/파인튜닝 측면의 long-context 접근. V4는 serving/inference 효율까지 강하게 겨냥.
Ring Attention (2023)blockwise transformer와 device ring으로 near-infinite context를 분산 처리.시스템 병렬화 기반 long-context. V4는 모델 구조 내부의 cache/attention 효율화와 연결.
StreamingLLM (2024)attention sink를 이용해 streaming 상황에서 안정적 추론을 유지.무한 스트리밍/긴 대화에서 “무엇을 보존할 것인가”라는 문제의 대표 연구.

3. KV Cache Compression / Long-context Inference

연구 흐름핵심 질문V4와의 관계
KV cache compression긴 context에서 key/value cache가 메모리를 선형으로 먹는 문제를 어떻게 줄일까?V4의 가장 중요한 실용 포인트. 1M context는 cache 비용을 줄이지 못하면 제품화가 어렵습니다.
DuoAttention류 head별 cache 전략모든 attention head가 같은 방식으로 긴 context를 볼 필요가 있는가?retrieval head와 non-retrieval head를 분리하는 아이디어는 V4의 hybrid attention 관점과 비교할 만합니다.
Squeezed/Compressed Attention 계열중요하지 않은 token/block을 압축하거나 선택적으로 attend할 수 있는가?V4의 CSA/HCA는 이 흐름을 모델 패밀리 핵심 구조로 끌어올린 사례입니다.

4. MoE / Open Model 경쟁 연구

GShard / Switch Transformer대규모 sparse expert model의 초기 대표 흐름. DeepSeekMoE가 비교 대상으로 삼는 계열.
Mixtral / sparse MoE open models오픈 모델에서 MoE가 성능-비용 tradeoff를 개선할 수 있음을 보여준 흐름.
Qwen / Llama / Mistral 계열DeepSeek V4가 경쟁해야 하는 오픈 모델 생태계. 성능뿐 아니라 라이선스, 배포, inference cost가 비교 기준.

5. Agentic Coding / Long-running Agent 평가

SWE-bench 계열실제 GitHub issue를 고치는 코딩 에이전트 평가. 긴 code context, test logs, tool-use가 중요합니다.
Repo-level code understanding단일 파일이 아니라 repository 전체 구조를 이해하는 능력.
Tool-use / browser / terminal agents긴 action trace를 유지하고 오류를 복구하는 능력. V4의 1M context가 체감될 가능성이 큰 영역.
