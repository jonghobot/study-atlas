Multi-Token Prediction: 한 번에 다음 여러 토큰을 예측하는 학습/추론 전략

MTP는 long-context 자체를 늘리는 기술은 아니지만, DeepSeek V3/V4 계열의 학습 신호, decoding throughput, speculative decoding 가능성과 연결되는 중요한 축입니다.

관련 논문 보기 리뷰 목록

문제의식

일반적인 language model은 매 위치에서 바로 다음 1개 token만 예측합니다. 이 방식은 안정적이지만, 모델이 더 먼 미래 token 구조를 직접 학습하는 신호가 약하고 autoregressive decoding은 token을 하나씩 생성해야 해서 latency가 커집니다.

방법

Multi-Token Prediction은 한 위치에서 다음 token 하나뿐 아니라 그 이후 여러 token을 함께 예측하도록 auxiliary objective나 별도 prediction head를 둡니다. DeepSeek 계열에서는 MTP module/block이 주 모델 뒤에 붙어 추가 미래 token 예측을 수행하는 방식으로 이해할 수 있습니다.

기여

MTP는 학습 시 더 풍부한 supervision을 제공하고, 추론 시에는 draft token을 여러 개 제안해 검증하는 speculative decoding류 최적화와 연결될 수 있습니다. 즉 모델 품질뿐 아니라 decoding throughput을 높이는 인프라적 의미가 있습니다.

DeepSeek V4와의 연결

DeepSeek V4의 핵심은 1M context와 hybrid attention이지만, 실제 agentic coding에서는 긴 context만큼 생성 속도도 중요합니다. MTP는 장기 tool-use, 코드 생성, 긴 답변 생성에서 token-by-token decoding 병목을 줄일 수 있는 보조 축입니다. V4 구조 설명에서 MTP block이 언급되는 만큼 관련 연구 맵에 포함하는 것이 자연스럽습니다.

한계와 읽을 때 던질 질문

MTP가 항상 품질을 높이는 것은 아닙니다. 여러 미래 token을 예측하는 objective가 주 next-token objective와 충돌할 수 있고, speculative decoding으로 이득을 보려면 draft token accept rate가 충분히 높아야 합니다. 또한 긴 context에서 중요한 병목인 KV cache memory 자체를 직접 줄이는 기술은 아닙니다.

이 페이지에서의 위치

MTP는 long-context extension보다는 decoding efficiency / training objective 축에 넣는 게 맞습니다. DeepSeekMoE가 모델 용량 대비 계산량, DSA/CSA/HCA가 긴 context attention 비용, KV cache compression이 memory 병목을 다룬다면, MTP는 생성 단계의 token throughput과 미래 token 학습 신호를 다룹니다.
