분야별 논문 리뷰

각 카드는 “Pop2Piano 후속 연구에 무엇을 줄 수 있는가?”를 중심으로 정리했습니다.

1. Direct Piano Cover Generation

2024
direct

PiCoGen: Generate Piano Covers with a Two-stage Approach

요약
Pop2Piano와 같은 piano cover generation 문제를 two-stage 방식으로 나누어 접근합니다.
방법
직접 end-to-end 생성보다 중간 표현/단계를 활용해 변환 난이도를 낮추는 방향입니다.
기여
Pop2Piano의 “한 번에 audio→cover” 프레임에 대한 대안적 설계를 제공합니다.
한계/연결
stage 분해가 controllability를 높이는지, error propagation을 만드는지 비교가 필요합니다.
2024
direct

PiCoGen2: Piano Cover Generation with Transfer Learning and Weakly Aligned Data

요약
strongly aligned paired data 의존성을 줄이기 위해 transfer learning과 weak alignment를 사용합니다.
방법
piano data로 pretrain한 뒤 약하게 정렬된 song-cover pair에 적응시키는 방향입니다.
기여
Pop2Piano의 가장 큰 병목인 dataset construction 문제를 직접 겨냥합니다.
한계/연결
weak alignment가 생성 품질/구조 보존에 주는 영향을 정량화하는 후속 실험이 중요합니다.
2024
direct

AMT-APC: Automatic Piano Cover by Fine-Tuning an Automatic Music Transcription Model

요약
기존 AMT 모델의 역량을 piano cover generation으로 이전하려는 접근입니다.
방법
transcription model을 fine-tuning해 cover 생성의 fidelity와 expressiveness 개선을 노립니다.
기여
AMT와 APC의 경계가 어디인지 실험적으로 묻는 좋은 baseline입니다.
한계/연결
transcription prior가 arrangement creativity를 제한할 가능성을 검증해야 합니다.

Pop2Piano 연결

좋은 transcription이 곧 좋은 arrangement는 아닙니다.

Pop2Piano 연결
MIDI decoder/tokenization 설계와 직접 연결됩니다.
한계
transcription task 중심이라 arrangement의 다중 정답성을 다루기 어렵습니다.
2021
AMT

MT3: Multi-Task Multitrack Music Transcription

요약
여러 악기 transcription을 text-like token sequence로 통합한 모델입니다.

Pop2Piano 연결
pop audio의 multi-instrument 정보를 piano arrangement로 압축하는 전처리/encoder 후보입니다.

3. Controllable / Symbolic Music Generation

2022
control

FIGARO: Generating Symbolic Music with Fine-Grained Artistic Control

요약
self-supervised description-to-sequence task로 symbolic music을 세밀하게 제어합니다.
기여
density, pitch/rhythm statistics 등 high-level descriptor 기반 control이 가능합니다.
Pop2Piano 연결
난이도, texture, section별 style을 조건화하는 후속 연구의 설계 참고점입니다.
한계
audio-conditioned cover generation으로 옮기려면 descriptor extraction과 alignment가 필요합니다.
2021/23
style

MuseMorphose: Full-Song and Fine-Grained Piano Music Style Transfer

요약
Transformer VAE로 full-song piano music의 style transfer를 수행합니다.
기여
style을 latent/control variable로 다루는 방식이 arranger style 연구에 유용합니다.
Pop2Piano 연결
“같은 pop song의 여러 cover style”을 모델링하는 후속 연구와 맞닿습니다.
한계
입력 audio의 구조/화성과 style transfer 목표를 동시에 만족시키는 문제는 더 어렵습니다.
2024
popular

BandCondiNet: Conditional Popular Music Generation with Multi-View Features

요약
popular music 생성을 multi-view condition과 parallel Transformer로 다룹니다.
기여
popular music의 구조, harmony, track interaction을 조건화하는 관점이 강합니다.
Pop2Piano 연결
pop song 구조를 piano cover로 보존하려면 multi-view condition 설계가 필요합니다.
한계
piano solo arrangement의 playability/voicing 문제는 별도 제약이 필요합니다.

4. Dataset / Evaluation / Representation

2024
data

PIAST: A Multimodal Piano Dataset with Audio, Symbolic and Text

요약
piano performance를 audio, symbolic, text modality로 묶은 dataset 방향입니다.
기여
text description과 symbolic/audio를 함께 쓰는 controllable generation 기반을 제공합니다.
Pop2Piano 연결
text prompt 기반 piano cover control 또는 평가 설명 데이터로 확장 가능합니다.
한계
pop-to-cover paired setting과 직접 같지는 않아 transfer 설계가 필요합니다.
2024
eval

Towards Musically Informed Evaluation of Piano Transcription Models

요약
note/frame IR metric을 넘어 articulation, dynamics, rhythmic precision 등 음악적 평가를 강조합니다.
기여
생성 결과를 음악적 속성별로 분해해 평가하는 언어를 제공합니다.
Pop2Piano 연결
cover generation 평가를 note overlap에서 playability/expressiveness로 확장할 수 있습니다.
한계
transcription 평가 중심이라 arrangement preference까지 포괄하려면 human eval이 필요합니다.
2024
style

Music Style Transfer with Time-Varying Inversion of Diffusion Models

요약
diffusion model inversion을 이용해 시간에 따라 변하는 음악 style transfer를 다룹니다.
기여
section별/시간별 style trajectory라는 관점이 있습니다.
Pop2Piano 연결
verse/chorus/bridge에 따라 다른 piano texture를 적용하는 cover generation과 연결됩니다.
한계
symbolic MIDI cover 생성과 직접 연결하려면 representation 변환이 필요합니다.

PiCoGen

단답 주관식

  1. PiCoGen이 piano cover generation을 two-stage로 나누는 이유는 무엇인가?

    변환 난이도를 낮추고 중간 표현을 활용하기 위해서다.

  2. Pop2Piano식 end-to-end 접근과 PiCoGen의 차이는 무엇인가?

    한 번에 audio→cover를 만들기보다 중간 단계를 거쳐 문제를 분해한다.

  3. two-stage 접근에서 생길 수 있는 대표 리스크는 무엇인가?

    앞 단계 오류가 뒤 단계로 전달되는 error propagation이다.

  4. PiCoGen이 후속 연구에 주는 설계 힌트는 무엇인가?

    중간 표현을 쓰면 controllability와 해석 가능성을 높일 수 있다는 점이다.

객관식

  1. PiCoGen의 핵심 접근은?

    two-stage piano cover generation

  2. two-stage 접근의 잠재 장점은?

    중간 표현으로 제어 가능성 증가

  3. two-stage 접근의 잠재 단점은?

    error propagation

  4. Pop2Piano와 비교할 때 PiCoGen이 던지는 질문은?

    end-to-end와 단계 분해 중 무엇이 나은가

PiCoGen2

단답 주관식

  1. PiCoGen2가 줄이려는 데이터 병목은 무엇인가?

    강하게 정렬된 pop-cover pair를 만들기 어렵다는 병목이다.

  2. weakly aligned data는 왜 유용한가?

    완벽히 맞춘 pair가 아니어도 더 많은 데이터를 활용할 수 있기 때문이다.

  3. transfer learning은 어떤 역할을 하는가?

    piano/music representation을 먼저 학습한 뒤 cover generation에 적응시키는 역할을 한다.

  4. weak alignment를 쓸 때 확인해야 할 품질 문제는 무엇인가?

    정렬 오차가 구조 보존과 생성 품질을 얼마나 해치는지 봐야 한다.

객관식

  1. PiCoGen2의 핵심 키워드는?

    transfer learning과 weak alignment

  2. weak alignment가 해결하려는 문제는?

    paired data 구축 비용

  3. transfer learning의 장점은?

    사전학습 지식을 cover generation에 이전

  4. PiCoGen2 평가에서 중요한 것은?

    약정렬 데이터가 품질에 주는 영향

AMT-APC

단답 주관식

  1. AMT-APC가 AMT 모델을 활용하는 이유는 무엇인가?

    AMT 모델이 오디오에서 note event를 뽑는 강한 prior를 갖고 있기 때문이다.

  2. transcription prior가 cover generation에 줄 수 있는 장점은 무엇인가?

    원곡의 음고·리듬 정보를 충실히 보존하는 데 도움을 줄 수 있다.

  3. transcription prior의 한계는 무엇인가?

    창의적 편곡, voicing, texture 변형을 제한할 수 있다.

  4. AMT와 arrangement generation은 어떻게 다른가?

    AMT는 들리는 음을 전사하는 문제이고 arrangement는 피아노적으로 재해석하는 문제다.

객관식

  1. AMT-APC의 출발점은?

    AMT 모델 fine-tuning

  2. AMT prior의 장점은?

    음악 정보 보존

  3. AMT prior의 위험은?

    편곡 창의성 제한

  4. AMT와 APC의 차이는?

    전사 vs 피아노 커버 재해석

Onsets and Frames

단답 주관식

  1. Onsets and Frames에서 onset prediction이 중요한 이유는 무엇인가?

    note 시작점을 정확히 잡아 frame prediction을 제약하기 위해서다.

  2. 이 모델이 Pop2Piano 평가에 주는 기준점은 무엇인가?

    원곡 음악 정보가 얼마나 보존됐는지 보는 transcription 기준점을 준다.

  3. note/frame F1만으로 cover 품질을 설명하기 어려운 이유는?

    playability, voicing, texture, expressiveness 같은 편곡 품질을 담지 못하기 때문이다.

  4. 좋은 transcription과 좋은 arrangement의 차이는 무엇인가?

    전자는 들리는 음의 정확한 복원이고 후자는 피아노로 자연스럽게 재구성하는 것이다.

객관식

  1. Onsets and Frames의 핵심은?

    onset과 frame의 dual objective

  2. Pop2Piano와 연결되는 지점은?

  3. note F1의 한계는?

    음악적 편곡 품질을 충분히 설명 못 함

  4. 좋은 cover 평가에 추가로 필요한 관점은?

    playability와 expressiveness

  1. time-varying style transfer의 의미는 무엇인가?

  2. diffusion inversion이 style transfer에 주는 역할은 무엇인가?

    기존 음악의 내용을 유지하면서 style 방향으로 변환하는 경로를 찾는 데 쓰인다.

  3. section별 piano texture 적용과 어떻게 연결되는가?

    verse, chorus, bridge마다 다른 texture나 난이도를 적용하는 cover generation과 연결된다.

  4. symbolic MIDI cover로 옮길 때 필요한 것은 무엇인가?

    audio/diffusion representation과 symbolic MIDI representation 사이의 변환 설계가 필요하다.
