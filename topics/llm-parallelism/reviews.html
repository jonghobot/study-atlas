LLM Parallelism 대표 자료와 연대

각 자료를 읽을 때 “무엇을 나누는가, 어떤 통신이 생기는가, 학습/서빙 중 어디에 더 중요한가”를 기준으로 보면 좋습니다. 아래는 2018년 pipeline 병렬화부터 2025년 serving scheduler/attention engine까지의 흐름입니다.

빠른 연대표

2018
pipeline

GPipe

Layer를 stage로 나누고 micro-batch로 pipeline을 채우는 출발점.

2019
tensor/memory

Megatron-LM, ZeRO

Transformer tensor parallelism과 optimizer/gradient/parameter sharding이 대형 LLM 학습의 기본 축으로 자리잡음.

2020–2021
MoE/3D

GShard, Switch Transformer, Megatron 3D Parallelism

Expert parallelism과 data × tensor × pipeline 조합이 trillion-scale 학습의 표준 언어가 됨.

2022
compiler

Alpa

Inter-operator와 intra-operator 병렬화를 자동 탐색/컴파일하는 방향을 제시.

2023
long context/serving

DeepSpeed Ulysses, PagedAttention/vLLM, SARATHI, Ring Attention, Splitwise

긴 sequence 병렬화와 LLM serving의 KV cache·prefill/decode 병목이 독립 연구 주제로 부상.

2024
disaggregation/4D

DistServe, USP, 4D Parallelism Estimator, SGLang

Prefill/decode 분리, sequence parallelism 통합, parallel config 추정, structured LLM program runtime으로 관심이 넓어짐.

2025
scheduler/kernel

FlashInfer, Online Scheduling with KV Cache Constraints

다양한 KV layout과 attention pattern을 다루는 serving kernel, KV cache 제약을 명시적으로 모델링한 scheduler가 중요해짐.

읽는 순서 추천

  1. 학습 기본기: GPipe → Megatron-LM → ZeRO → Megatron 3D Parallelism
  2. MoE 확장: GShard → Switch Transformer → expert parallelism/load balancing 자료
  3. Long context: DeepSpeed Ulysses → Ring Attention → USP
  4. Serving: PagedAttention/vLLM → SARATHI → Splitwise/DistServe → FlashInfer → KV-aware scheduling
  5. 자동화/운영: Alpa → 4D memory estimator → 실제 cluster config tuning

제안: 다음에 만들면 좋은 하위 페이지

  • vLLM/PagedAttention 상세 리뷰: LLM serving을 이해하는 데 가장 실용적인 출발점이라 우선순위가 높습니다.
  • DistServe vs Splitwise 비교: prefill/decode disaggregation이 앞으로 serving architecture의 핵심 축이라 비교 페이지가 있으면 좋습니다.
  • Ulysses vs Ring Attention vs USP: long-context parallelism을 한 장의 표와 예시로 정리하면 DeepSeek/long-context 주제와도 잘 연결됩니다.
  • 실습 노트: “Llama 계열 70B를 8×GPU에 올릴 때 TP/PP/FSDP를 어떻게 고를까?” 같은 config 선택 예제를 추가하면 학습 페이지가 훨씬 실전적이 됩니다.
