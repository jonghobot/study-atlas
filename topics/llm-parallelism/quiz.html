LLM Parallelism 이해 확인

Concepts 페이지를 읽은 뒤 핵심 병렬화 축을 구분할 수 있는지 확인합니다.

핵심 질문

  1. Tensor parallelism과 pipeline parallelism은 각각 무엇을 나누는가?

    Tensor parallelism은 layer 내부 행렬 곱/weight 차원을, pipeline parallelism은 layer 묶음/stage를 나눈다.

  2. ZeRO/FSDP가 data parallel의 어떤 낭비를 줄이는가?

    각 복제본이 parameter, gradient, optimizer state를 모두 들고 있는 메모리 낭비를 shard로 줄인다.

  3. MoE에서 expert parallelism의 핵심 리스크는?

    token routing all-to-all 통신과 expert load imbalance다.

  4. LLM serving에서 KV cache 관리가 중요한 이유는?

    요청별 context 길이에 비례해 KV cache가 커지고, fragmentation과 memory bandwidth가 throughput/tail latency를 좌우하기 때문이다.
