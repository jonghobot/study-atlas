Topic 03 · LLM Systems

LLM Parallelism: 거대 모델을 나눠 학습하고 서빙하는 법

거대 LLM은 알고리즘만으로 커지지 않습니다. 파라미터, activation, sequence, expert, KV cache, request batch를 여러 GPU와 노드에 어떻게 나눌지 설계해야 학습과 서빙이 현실화됩니다.

Training

Data, tensor, pipeline, ZeRO/FSDP를 조합해 모델 상태와 activation을 분산합니다.

Long Context

Sequence/context parallelism과 attention kernel 최적화로 긴 prompt의 prefill 부담을 다룹니다.

Serving

Continuous batching, PagedAttention, chunked prefill, prefill/decode disaggregation으로 KV cache와 tail latency를 관리합니다.

연대별 핵심 흐름

2018–2019

Pipeline/Tensor/Memory sharding의 기초

GPipe, Megatron-LM, ZeRO가 “layer를 나누기”, “행렬 곱을 나누기”, “모델 상태를 shard하기”라는 기본 언어를 만들었습니다.

2020–2022

MoE와 자동 병렬화

GShard/Switch Transformer가 expert parallelism을 대중화했고, Alpa는 병렬화 config를 사람이 손으로만 고르지 않는 방향을 제시했습니다.

2023

Long-context와 serving 병목의 독립

Ulysses/Ring Attention은 sequence를 나누는 법을, vLLM/PagedAttention과 SARATHI는 KV cache와 prefill/decode scheduling을 전면에 올렸습니다.

2024–2025

Disaggregated serving과 KV-aware runtime

DistServe, USP, 4D memory estimator, FlashInfer, KV-aware online scheduling처럼 “어떤 parallelism을 고를지”와 “어떻게 runtime에서 운영할지”가 최신 쟁점입니다.

내 제안

이 주제는 단순 개념 페이지보다 학습 병렬화, long-context 병렬화, serving 병렬화 세 갈래로 키우는 게 좋겠습니다. 특히 다음 상세 리뷰는 가치가 큽니다.

  • vLLM/PagedAttention: 실무 serving 이해의 중심.
  • DistServe vs Splitwise: prefill/decode 분리 아키텍처 비교.
  • Ulysses vs Ring Attention vs USP: long-context sequence parallelism 비교.
  • 4D parallelism config 실습: GPU 수와 모델 크기에 따라 TP/PP/DP/CP를 고르는 예제.
